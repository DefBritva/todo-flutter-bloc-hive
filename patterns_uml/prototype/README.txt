Данный паттерн выбран по причинам:
    Меньше повторяющегося кода инициализации объектов.
    Ускоряет создание объектов

    В проекте используется библиотека flutter bloc. Bloc отправляет в ui текущее состояние (state).
    Сам класс состояния является неизменяемым, чтобы отправить новое состояние в ui, создаётся новый экземпляр
    состояния. Зачастую приходится изменять не все поля (переменные) состояния, а лишь некоторые из них,
    использование метода clone() с параметрами позволяет не писать лишний код. Например состояние хранит
    экземпляр класса Note (заметку), заметка имеет четыре поля. В новом состоянии изменяется только одно из этих
    полей. (Мы имеем доступ к прошлому состоянию, его полям) Вместо того чтобы создавать новый экземпляр класса
    состояния и передавать все старые поля, в новое состояние для отправки в ui,
    удобно использовать метод clone(var par) для клонирования прошлого состояния с измением переданного параметра
