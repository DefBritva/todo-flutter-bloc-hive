Приложение работает следующим образом:
    Оно разделено на 3 уровня:
        - Уровень представления (UI - пользовательский интерфейс)
        - Уровень BLoC (business logic component)
        - Уровень данных

    В свою очередь BLoC содержит в себе экземпляры отдельных классов логики приложения (Services) и использует их.
    Типичный сценарий работы приложения состоит в следующем:
        1. Пользователь совершает действие в UI => из UI в BLoC отправляется конкретное событие (Event)
        2. BLoC обрабатывает это событие и отправляет запрос к базе данных и получает ответ.
        3. BLoC отравляет новое состояние приложения в UI => UI перестраивается на основе нового состояния.
